# DesignPocket 仕様書 v1.0

## アプリ概要

| 項目     | 内容                           |
| ------ | ---------------------------- |
| アプリ名   | DesignPocket（デザインぽけっと）       |
| 種別     | Webアプリ（ローカル動作）               |
| バージョン  | v1.0                         |
| 技術スタック | HTML / CSS / JavaScript（バニラ） |
| データ永続化 | localStorage                 |
| 外部依存   | なし（ライブラリ不使用）                 |

### コンセプト

開発者のためのUIインスピレーション管理Webアプリ。スクショやURLを「どのアプリのどの画面に使いたいか」と紐づけてギャラリー形式で管理する。Pinterestのような見た目と、開発者特化の整理機能を両立する。ブラウザで開くだけで動作し、インストール不要。

## v1 スコープ

- ギャラリー表示（グリッド）
- アイデア登録
- アプリ別管理

## 画面構成

### ナビゲーション（上部タブ）

```
[ ギャラリー ] [ アプリ別 ] [ ＋追加 ]
```

### ギャラリー画面

- 2〜3カラムのMasonryグリッド表示
- 画像またはURLカードをタイル状に表示
- フィルタバー：全て / アプリ名 / タグ
- カードクリックで詳細モーダル表示

### アプリ別画面

- 登録済みアプリをカード一覧表示（絵文字＋名前＋件数）
- アプリカードクリックで紐づいたアイデア一覧表示
- アプリの追加・削除

### 追加画面（モーダルまたは別セクション）

- 画像アップロード（ファイル選択）
- 参考URL入力
- タイトル（必須）
- タグ入力（カンマ区切り）
- 紐づけるアプリ選択（プルダウン）
- 画面名入力（任意）
- 実装メモ（任意）

## データ構造

### AppProject（アプリ）

```javascript
{
  id: string,        // UUID
  name: string,      // アプリ名
  emoji: string,     // 絵文字アイコン
  createdAt: string  // ISO日付
}
```

### DesignIdea（アイデア）

```javascript
{
  id: string,              // UUID
  title: string,           // タイトル（必須）
  imageDataUrl: string,    // Base64画像（任意）
  referenceUrl: string,    // 参考URL（任意）
  tags: string[],          // タグ配列
  appId: string | null,    // 紐づけるアプリID
  screenName: string,      // 画面名（任意）
  note: string,            // 実装メモ（任意）
  createdAt: string        // ISO日付
}
```

### localStorage キー設計

```
designpocket_apps    → AppProject[]のJSON
designpocket_ideas   → DesignIdea[]のJSON
```

## UI仕様

### カラー・トーン

- ベース：白 / ライトグレー
- アクセント：インディゴ系（#6366f1）
- フォント：システムフォント

### カードデザイン

- 角丸（border-radius: 12px）
- ドロップシャドウ
- 画像がある場合：上部に画像、下部にタイトル・タグ
- 画像がない場合：URLアイコン＋タイトル表示

### レスポンシブ

- 1200px以上：3カラム
- 768px〜1199px：2カラム
- 767px以下：1カラム

## Codex評価観点（v1）

| カテゴリ      | 評価ポイント                   |
| --------- | ------------------------ |
| データ管理     | localStorageの読み書きに問題はないか |
| フィルタリング   | タグ・アプリ別フィルタのロジックは正確か     |
| 画像処理      | Base64変換・表示の実装に問題はないか    |
| UI構成      | HTML/CSSの構造が整理されているか     |
| エラーハンドリング | ストレージ上限・画像読み込み失敗の考慮      |
| 状態管理      | DOMの更新ロジックに副作用・バグはないか    |
